# MarkFlow Best Practices (트러블슈팅 및 모범 사례)

본 문서는 프로젝트 진행 중 발생한 주요 이슈와 이를 해결하며 정립한 베스트 프랙티스를 기록합니다.

## 1. 이메일 전송 (Nodemailer) 타임아웃 및 포트 설정
- **이슈**: 배포 환경(Railway)에서 구글 SMTP(포트 465) 접근 시 IP 차단이나 방화벽 문제로 인해 전송이 2분 이상 무한 지연되는 현상 발생.
- **해결 및 프랙티스**:
  - 포트를 **587(STARTTLS)**로 변경하여 클라우드 서버의 SMTP 465 포트 차단을 우회.
  - 전송을 무한정 대기(비동기)하지 않고, 동기식(`await`)으로 처리하되 **5초의 타임아웃(`connectionTimeout: 5000`)** 방어막을 설정.
  - 없는 이메일을 입력하거나 서버 에러 발생 시 2분을 기다리는 대신 5초 만에 실패를 감지하여 프론트엔드에 즉시 에러("이메일 발송에 실패했습니다")를 던져주는 UX 확보.
  - 프론트엔드에서는 남은 시간을 시각적으로 보여주는 **3분 타이머(03:00)**를 배치하여, 타이머 만료 전까지는 자체적으로 검증하고 재전송을 유도하는 방식을 채택.

## 2. 409 Conflict (다른 기기 로그인) 시 세션 쿠키 초기화
- **이슈**: 사용자가 로그인 화면으로 튕겨 나간 후 새로고침(F5)을 하면, 브라우저에 남아있는 유효하지 않은 Refresh 쿠키 때문에 계속 `/auth/refresh` API가 호출되고 409 에러를 뿜으며 무한 루프(재발)에 빠짐.
- **해결 및 프랙티스**:
  - 백엔드(`/auth/refresh`)에서 `409 Conflict` 예외가 발생할 경우, 예외를 던지기 직전에 응답 헤더에 `res.clearCookie()`(Set-Cookie 만료)를 강제로 쏴줌.
  - 클라이언트 브라우저는 409 응답과 함께 쿠키를 깔끔하게 삭제하게 되므로, 이후 새로고침을 해도 쿠키가 없어 깔끔하게 비로그인 상태로 머무르게 됨.

## 3. 웹소켓 실시간 동기화 에코(Echo) 방지
- **이슈**: 내가 노드를 수정했을 때, 서버가 브로드캐스팅한 이벤트를 다시 내가 수신하여 로컬 상태가 한 번 더 덮어씌워지는 깜빡임(Echo) 현상 발생.
- **해결 및 프랙티스**:
  - 소켓 이벤트 페이로드에 이벤트를 발생시킨 `triggerUserId`를 포함.
  - 클라이언트(프론트엔드)에서는 메시지를 수신했을 때 `message.triggerUserId === currentUser.id` 이면 자신이 보낸 액션으로 간주하고 상태 업데이트를 무시(Ignore)하도록 처리.

## 4. OTP 재전송 쿨타임 처리
- **이슈**: 사용자가 3분을 기다리지 않고 이메일을 재전송할 때 기존 코드가 만료되지 않았다고 막히는 문제.
- **해결 및 프랙티스**:
  - 재전송 요청 시 기존 DB에 남아있던 OTP 레코드를 조건 없이 `deleteMany`로 삭제하고 새 OTP를 생성.
  - 사용자는 3분 제한에 걸리지 않고 즉시 새 코드를 받을 수 있음.
