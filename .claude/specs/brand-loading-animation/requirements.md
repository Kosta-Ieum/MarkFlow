# 브랜드 로딩 애니메이션 — 요구사항

## 1. Overview

### 문제 정의
- 부팅 로딩(`BootLoading`)이 "불러오는 중…" 텍스트 한 줄뿐이라 밋밋하고 브랜드 정체성이 없다.
- 캔버스 진입 시 `canvasStore.isLoading`은 존재하지만 이를 표시하는 UI가 없어, 데이터 도착 전까지 빈 캔버스가 노출된다.

### 목표
- 파비콘과 동일한 "M 노드 그래프" 아이콘을 활용한 로딩 애니메이션 컴포넌트를 만들고, 부팅 로딩과 캔버스 진입 로딩에 적용한다.
- 애니메이션: 엣지가 순서대로 그려지고 노드가 차례로 점등하며 M이 완성되는 사이클 반복.

### Non-goals
- 프로젝트 목록 스켈레톤 카드 교체 (현행 유지).
- 버튼·모달 등 소규모 인라인 로딩 인디케이터 교체.
- 애니메이션 라이브러리 도입 (순수 CSS/SVG로 구현).

### 성공 기준
- 새로고침/첫 진입 시 "불러오는 중…" 텍스트 대신 M 애니메이션이 보인다.
- 캔버스 데이터 로딩 중 빈 캔버스 대신 M 애니메이션 오버레이가 보인다.
- 접근성(스크린리더 안내·모션 최소화 설정) 유지.

## 2. 요구사항

### R1. 브랜드 로딩 컴포넌트

**유저스토리:** 사용자로서, 로딩 중임을 브랜드 아이콘 애니메이션으로 인지하고 싶다.

**인수 조건:**
- R1.1 WHEN 로딩 컴포넌트가 렌더되면, THE SYSTEM SHALL 파비콘과 동일한 도형(노드 5개 + 엣지 4개의 M)을 SVG로 표시한다.
- R1.2 WHILE 로딩 상태가 지속되는 동안, THE SYSTEM SHALL 엣지 드로잉 → 노드 점등 순서로 M이 완성되는 애니메이션을 반복 재생한다.
- R1.3 WHEN 애니메이션 사이클이 끝나면, THE SYSTEM SHALL 완성된 M을 짧게 유지한 뒤 다음 사이클을 시작한다.
- R1.4 IF 사용자 OS가 모션 최소화(`prefers-reduced-motion: reduce`)로 설정되어 있으면, THEN THE SYSTEM SHALL 드로잉 애니메이션 대신 정적인 아이콘(완성된 M)을 표시한다.
- R1.5 WHEN 로딩 컴포넌트가 렌더되면, THE SYSTEM SHALL 스크린리더용 텍스트(예: "불러오는 중")를 시각적으로 숨긴 채 제공한다 (`role="status"` + `aria-live` 유지).
- R1.6 WHILE 로딩 화면이 표시되는 동안, THE SYSTEM SHALL 화면에 시각적 텍스트를 표시하지 않는다 (아이콘만).

### R2. 부팅 로딩 적용

**유저스토리:** 사용자로서, 새로고침·첫 진입 시 세션 복원을 기다리는 동안 브랜드 로딩 화면을 보고 싶다.

**인수 조건:**
- R2.1 WHILE `authStore.isBootstrapping`이 true인 동안, THE SYSTEM SHALL 기존 "불러오는 중…" 텍스트 대신 R1 컴포넌트를 화면 중앙에 표시한다.
- R2.2 WHEN 부팅 복원이 완료되면, THE SYSTEM SHALL 기존과 동일한 라우팅 동작(인증 시 원래 화면, 미인증 시 /login)을 유지한다.

### R3. 캔버스 진입 로딩 적용

**유저스토리:** 사용자로서, 캔버스 데이터를 불러오는 동안 빈 캔버스 대신 로딩 중임을 보고 싶다.

**인수 조건:**
- R3.1 WHILE `canvasStore.isLoading`이 true인 동안, THE SYSTEM SHALL 캔버스 영역 위에 R1 컴포넌트를 포함한 오버레이를 표시한다.
- R3.2 WHEN 캔버스 데이터 로드가 완료되면, THE SYSTEM SHALL 오버레이를 제거하고 캔버스 조작을 방해하지 않는다.
- R3.3 WHILE 오버레이가 표시되는 동안, THE SYSTEM SHALL 캔버스 노드·엣지에 대한 포인터 조작을 차단한다 (로딩 중 조작으로 인한 어긋남 방지).
