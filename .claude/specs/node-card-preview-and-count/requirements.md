# 노드 카드 md 미리보기 개선 + 생성 번호 고유화 — 요구사항

## 1. Overview

### 문제 정의
1. **펼친 카드 배경 깨짐**: 캔버스 노드 카드를 펼치면 본문 마크다운이 `MDEditor.Markdown`으로 렌더되는데, 캔버스에는 컬러 모드가 지정돼 있지 않아 OS가 다크모드인 사용자에겐 **흰 카드 안에 검은 배경 본문**이 렌더된다(노드 에디터 화면은 `data-color-mode="light"`를 지정해 정상).
2. **카드가 무한정 길어짐**: 펼침 상태는 높이 제한이 없어 본문이 길면 카드가 캔버스를 세로로 덮는다.
3. **새 노드 번호 중복**: 새 노드 기본 이름이 `새 노드 (현재 노드 수+1)`이라, 삭제 후 생성하면 기존 노드와 같은 번호가 다시 부여된다(예: 3개 중 1개 삭제 후 생성 → "새 노드 3" 두 개).

### 목표
- 펼친 카드 본문을 **카드와 동일한 라이트 배경**으로, 에디터 미리보기와 정합한 서식으로 렌더한다.
- 펼친 카드에 **높이 상한**을 두고, 넘치는 본문은 잘림을 시각적으로 표시한다(전체 내용은 기존 더블클릭 → 에디터 진입으로).
- 새 노드 기본 이름 번호를 **고유하게** 부여한다(삭제 후에도 중복 없음).

### Non-goals (이번 범위 제외)
- 서버 측 생성 카운터 영속(스키마·BE 변경) — 클라이언트 계산으로 충분.
- 노드 개수 통계 표시(사이드바/프로젝트 목록의 개수 카운터).
- 노드 에디터 화면의 렌더 변경.
- 접힌 카드 미리보기(첫 줄 26자) 방식 변경.
- 호버 팝업 미리보기.

### 성공 기준
- OS 다크모드에서도 펼친 카드 본문이 흰 배경·정상 서식으로 보인다.
- 긴 본문을 펼쳐도 카드가 상한 높이를 넘지 않고, 잘렸음이 시각적으로 드러난다.
- 노드를 삭제한 뒤 새로 만들어도 화면·휴지통의 기존 노드와 기본 이름 번호가 겹치지 않는다.

## 2. 요구사항

### R1. 펼친 카드 라이트 테마 고정

**유저스토리:** 다크모드 OS 사용자로서, 카드와 본문의 배경이 어긋나 보이지 않도록, 펼친 본문이 항상 카드와 같은 라이트 테마로 렌더되길 원한다.

**인수 조건:**
- R1.1 WHEN 노드 카드를 펼치면, THE SYSTEM SHALL OS/브라우저 다크모드 여부와 무관하게 본문을 라이트 배경(카드 배경과 동일 계열)으로 렌더한다.
- R1.2 THE SYSTEM SHALL 제목·리스트·체크박스·코드블록 등 마크다운 서식을 에디터 미리보기와 동일한 렌더러 기반으로 유지한다.

### R2. 펼친 카드 높이 상한 + 잘림 표시

**유저스토리:** 캔버스 사용자로서, 본문이 긴 노드를 펼쳐도 캔버스 배치가 망가지지 않도록, 카드 높이에 상한이 있길 원한다.

**인수 조건:**
- R2.1 WHEN 펼친 본문의 렌더 높이가 상한(설계에서 확정)을 초과하면, THE SYSTEM SHALL 상한까지만 표시하고 하단에 잘림 표시(말줄임·페이드 등)를 렌더한다.
- R2.2 WHEN 본문이 상한 이하이면, THE SYSTEM SHALL 잘림 표시 없이 전체를 표시한다.
- R2.3 THE SYSTEM SHALL 잘린 상태에서도 기존 전체 열람 경로(더블클릭 → 노드 에디터)를 그대로 제공한다.
- R2.4 WHEN 표·긴 코드 등 카드 폭(186px)을 넘는 콘텐츠가 있으면, THE SYSTEM SHALL 카드 폭 밖으로 넘치지 않게 처리한다(내부 잘림 또는 가로 스크롤).

### R3. 새 노드 기본 이름 번호 고유화

**유저스토리:** 편집자로서, 노드를 지웠다 만들어도 이름이 겹치지 않도록, 새 노드 번호가 고유하게 매겨지길 원한다.

**인수 조건:**
- R3.1 WHEN 새 노드를 생성하면, THE SYSTEM SHALL 기본 이름을 "새 노드 N"으로 부여하되 N은 현재 캔버스와 휴지통에 존재하는 "새 노드 <숫자>" 제목들의 최대 번호 + 1로 계산한다.
- R3.2 WHEN 일부 노드를 삭제(휴지통 이동 포함)한 뒤 새 노드를 생성하면, THE SYSTEM SHALL 살아있는 노드·휴지통 노드와 번호가 중복되지 않는 이름을 부여한다.
- R3.3 THE SYSTEM SHALL 사용자가 지정한 제목(기본 이름 패턴이 아닌 제목)에는 어떤 영향도 주지 않는다.
