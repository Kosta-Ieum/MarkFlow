# 캔버스 Undo/Redo — 요구사항

## 1. Overview

### 문제 정의
캔버스에서 노드를 실수로 옮기거나 지우고, 엣지를 잘못 잇거나 끊고, 노드 내용을 잘못 고쳐도 **되돌릴 방법이 없다.** 실시간 협업이라 실수가 즉시 다른 참가자에게도 반영·저장되므로, 되돌리기(undo)/다시 실행(redo) 수단의 부재가 특히 아프다.

### 목표
편집 권한이 있는 사용자가 **자신의 최근 캔버스 동작을 순서대로 되돌리고 다시 실행**할 수 있게 한다. 진입점은 캔버스 하단 도구모음 버튼과 키보드 단축키(Ctrl/Cmd+Z, Ctrl/Cmd+Shift+Z). 되돌린 결과는 일반 편집과 동일하게 다른 참가자에게 실시간 반영되고 서버에 저장된다.

### Non-goals (이번 범위 제외)
- **서버 영속**: 새로고침·재접속 후에도 undo 히스토리를 유지 → 후속 과제. 이번엔 세션(클라이언트) 메모리만.
- **글로벌/타인 동작 undo**: 남이 한 동작을 되돌리는 기능. 이번엔 per-user(내 동작만).
- **노드 편집기 내부 키 단위 텍스트 undo**: 편집기 textarea 안의 타이핑 되돌리기는 브라우저 native undo가 담당. 캔버스 undo는 "저장(변경 확정) 1회"를 하나의 단위로 다룬다.
- **접기/펼치기·뷰포트(줌/팬)** 되돌리기.
- OT/CRDT 기반 동시편집 병합.

### 성공 기준
- 대상 6종 연산(노드 생성·삭제·이동, 엣지 연결·해제, 노드 제목·내용 수정)에 대해 undo/redo가 정확히 동작한다.
- 되돌린 결과가 다른 참가자에게 실시간 반영되고 서버에 저장된다.
- VIEWER는 undo/redo를 쓸 수 없다(버튼 비활성화 + 서버 최종 가드).
- 되돌릴 대상이 이미 바뀐(삭제·잠금 등) 상황에서 앱이 깨지지 않고 안전하게 스킵·안내한다.

## 2. 요구사항

### R1. Undo/Redo 기본 동작 (버튼 + 단축키)

**유저스토리:** 편집자로서, 방금 한 캔버스 동작을 즉시 되돌리기 위해, 하단 도구모음 버튼과 키보드 단축키로 undo/redo를 실행하고 싶다.

**인수 조건:**
- R1.1 WHEN 사용자가 Undo 버튼을 누르거나 Ctrl/Cmd+Z를 입력하면, THE SYSTEM SHALL 그 사용자의 가장 최근 되돌리기 가능한 동작을 역연산으로 되돌린다.
- R1.2 WHEN 사용자가 Redo 버튼을 누르거나 Ctrl/Cmd+Shift+Z를 입력하면, THE SYSTEM SHALL 직전에 되돌린 동작을 다시 적용한다.
- R1.3 WHILE undo 스택이 비어 있으면, THE SYSTEM SHALL Undo 진입점을 비활성화하고 단축키 입력을 무시한다. (redo 스택도 동일)
- R1.4 WHEN 노드 편집기 텍스트 입력에 포커스가 있는 동안 Ctrl/Cmd+Z가 입력되면, THE SYSTEM SHALL 캔버스 undo를 실행하지 않고 편집기의 텍스트 undo에 양보한다.

### R2. 되돌리기 대상 연산

**유저스토리:** 편집자로서, 구조·내용 변경 실수를 되돌리기 위해, 아래 동작들이 undo/redo 대상이 되길 원한다.

**인수 조건:**
- R2.1 WHEN 사용자가 노드를 생성한 뒤 undo하면, THE SYSTEM SHALL 그 노드를 삭제(휴지통 이동)한다.
- R2.2 WHEN 사용자가 노드를 삭제한 뒤 undo하면, THE SYSTEM SHALL 그 노드를 삭제 직전 상태로 복원한다(기존 restore 재사용).
- R2.3 WHEN 사용자가 노드를 이동한 뒤 undo하면, THE SYSTEM SHALL 그 노드를 이동 직전 좌표로 되돌린다.
- R2.4 WHEN 사용자가 엣지를 연결한 뒤 undo하면, THE SYSTEM SHALL 그 엣지를 삭제한다.
- R2.5 WHEN 사용자가 엣지를 삭제한 뒤 undo하면, THE SYSTEM SHALL 같은 source·target으로 엣지를 복원한다.
- R2.6 WHEN 사용자가 노드 제목 또는 마크다운 내용을 저장(변경 확정)한 뒤 undo하면, THE SYSTEM SHALL 해당 값을 변경 직전 값으로 되돌린다.
- R2.7 THE SYSTEM SHALL 각 대상 동작을 하나의 되돌리기 단위(step)로 다룬다 — 드래그 1회 = 이동 1 step, 저장 1회 = 내용 수정 1 step.
- R2.8 THE SYSTEM SHALL 접기/펼치기와 뷰포트(줌/팬) 변경을 되돌리기 대상에서 제외한다.

### R3. Per-user 범위 (내 동작만)

**유저스토리:** 참가자로서, 남의 작업을 건드리지 않고 내가 한 것만 되돌리기 위해, undo가 내 동작에만 적용되길 원한다.

**인수 조건:**
- R3.1 THE SYSTEM SHALL 각 클라이언트에서 그 사용자 본인이 수행한 동작만 undo 스택에 기록한다.
- R3.2 WHEN 다른 참가자의 동작을 원격 수신하면, THE SYSTEM SHALL 그 동작을 자신의 undo 스택에 넣지 않는다.
- R3.3 WHEN 사용자가 undo/redo를 실행하면, THE SYSTEM SHALL 그 역연산/재연산 결과를 일반 편집과 동일하게 다른 참가자에게 실시간 전파하고 서버에 저장한다.

### R4. 실시간·영속 정합성

**유저스토리:** 참가자로서, 되돌린 결과가 모두에게 동일하게 보이도록, undo/redo가 기존 협업 경로를 그대로 타길 원한다.

**인수 조건:**
- R4.1 WHEN undo/redo가 적용되면, THE SYSTEM SHALL 변경을 기존 협업 경로(소켓 emit + 서버 영속 + ActivityLog 기록)를 통해 반영한다.
- R4.2 WHEN undo/redo로 발생한 변경을 원격 수신하면, THE SYSTEM SHALL 수신 클라이언트가 그것을 자신의 undo 스택에 넣지 않는다(에코 루프 방지).

### R5. 충돌·무효 처리

**유저스토리:** 참가자로서, 되돌릴 대상을 다른 사람이 이미 바꿨을 때도 앱이 깨지지 않도록, 안전한 실패 처리를 원한다.

**인수 조건:**
- R5.1 IF undo/redo 대상 노드·엣지가 이미 삭제되었거나 존재하지 않으면, THEN THE SYSTEM SHALL 그 step을 적용하지 않고 스택에서 제거한 뒤 사용자에게 되돌릴 수 없음을 알린다.
- R5.2 IF undo 대상 노드가 다른 사용자에 의해 편집 잠금(soft-lock) 상태이면, THEN THE SYSTEM SHALL 그 step 적용을 거부하고 사용자에게 알린다.
- R5.3 IF 서버가 undo/redo로 발생한 변경을 권한·검증 이유로 거부하면, THEN THE SYSTEM SHALL 로컬 상태를 서버 기준으로 회복시키고 사용자에게 실패를 알린다.

### R6. 스택 수명 · redo 무효화

**유저스토리:** 사용자로서, 예측 가능한 되돌리기 범위를 위해, 세션 단위의 히스토리와 표준적인 redo 무효화를 원한다.

**인수 조건:**
- R6.1 THE SYSTEM SHALL undo/redo 스택을 현재 캔버스 세션의 메모리에만 유지한다.
- R6.2 WHEN 사용자가 캔버스를 벗어나거나 페이지를 새로고침하면, THE SYSTEM SHALL undo/redo 스택을 초기화한다.
- R6.3 WHEN 사용자가 새로운 되돌리기 대상 동작을 수행하면, THE SYSTEM SHALL redo 스택을 비운다.
- R6.4 THE SYSTEM MAY undo 스택 크기에 상한(최근 N개)을 두어 메모리 사용을 제한한다. (구체값은 설계에서 확정)

### R7. 권한 (VIEWER)

**유저스토리:** 뷰어로서, 편집 권한이 없으므로, undo/redo가 제공되지 않길(혼란 방지) 원한다.

**인수 조건:**
- R7.1 WHILE 사용자의 역할이 VIEWER이면, THE SYSTEM SHALL undo/redo 진입점을 비활성화하고 단축키를 무시한다.
- R7.2 THE SYSTEM SHALL 편집 권한의 최종 가드를 서버(REST·Socket)에 둔다 — 프론트 비활성화는 UX 보조일 뿐이다.

### R8. UI 배치

**유저스토리:** 사용자로서, 익숙한 위치에서 되돌리기를 쓰기 위해, 하단 도구모음의 undo/redo 버튼을 원한다.

**인수 조건:**
- R8.1 THE SYSTEM SHALL Undo·Redo 버튼을 캔버스 하단 도구모음(기존 줌 컨트롤 영역)에 배치한다.
- R8.2 THE SYSTEM SHALL 각 버튼의 가능/불가능 상태를 비활성화 등으로 시각적으로 표시한다.
