# Markflow — 흩어진 아이디어를, 흐름으로

마크다운 노드 기반 실시간 협업 캔버스
3인 / 4주 · React · Socket.io · PostgreSQL

🖼️ 이미지: 점격자 무한 캔버스 위에 컬러 마크다운 노드들이 곡선으로 연결된 미니멀 일러스트, 웜 오프화이트 배경에 그린 포인트

---

# 우리가 겪은 문제

- 화이트보드 스티키 노트는 정보 밀도가 낮아 정리된 문서로 잇기 어렵다
- 문서 도구는 내용은 풍부하지만 아이디어의 '흐름'을 시각화하기 어렵다
- 실시간 협업 시 누가 무엇을 작업 중인지 보이지 않는다

핵심: 브레인스토밍의 가벼움과 문서의 밀도를 한 화면에서 오가는 도구가 비어 있다

---

# 인사이트, 그리고 해결

| 도구 | 정보 밀도 | 흐름 시각화 | 실시간 협업 |
| --- | --- | --- | --- |
| FigJam / Miro | 낮음 | 좋음 | 좋음 |
| Notion / Docs | 높음 | 약함 | 보통 |
| Markflow | 높음 | 좋음 | 좋음 |

해결: 무한 캔버스 위에서 .md 노드를 작성·연결하고, 멀티커서·채팅으로 함께 정리한다
→ 마크다운 노드로 밀도를, 노드 연결로 흐름을, 소켓 동기화로 협업을 동시에

---

# 핵심 ① 마크다운 노드

- 노드 = .md 파일. 접으면 제목+요약 한 줄, 펼치면 코드 블록까지 담는 상세 문서
- 5가지 타입: 아이디어 · 문서 · 할 일 · 결정 · 데이터
- 더블클릭 → 전체화면 마크다운 에디터(작성 / 미리보기 2-pane)

스티키 메모보다 깊게, 문서보다 가볍게

🖼️ 이미지: 접힌 노드 카드와 펼쳐진 마크다운 노드 카드를 나란히 비교

---

# 핵심 ② 흐름 연결

- 두 노드를 엣지로 연결 → 화살표 방향이 곧 아이디어의 흐름
- 흩어진 메모가 플로우차트가 된다
- 무한 캔버스: 팬 · 줌 · 미니맵 · 전체보기

🖼️ 이미지: 킥오프→요구사항→채택 결정으로 이어지는 노드 플로우차트

---

# 핵심 ③ 실시간 협업 (하이라이트)

- 멀티커서 — 누가 어디서 작업 중인지 실시간 표시
- 소프트 락 — "OO 편집 중" 표시로 한 노드 한 명 편집, 충돌 회피
- 캔버스 채팅 — 같은 캔버스에서 바로 대화
- 끊김 재접속 시 상태 자동 재동기화

Figma처럼, 같은 보드에서 함께 쓰고 바로 이야기한다

---

# 데모 시나리오

1. 회원가입 / 로그인 (JWT)
2. 프로젝트 생성 → 캔버스 자동 생성
3. 마크다운 노드 작성 · 이동 · 연결
4. 2명 동시 접속 → 멀티커서 · 소프트 락 · 채팅
5. 노드 삭제 → 휴지통 → 복구
6. 변경 히스토리 타임라인 확인

목표: 핵심 흐름 오류 없이, 변경 약 1초 이내 반영

---

# 기술 스택

- 프론트: React · TypeScript · Vite · React Flow · Zustand · Tailwind
- 실시간: Socket.io 직접 구현 (Liveblocks 차선)
- 백엔드: Node.js · Express · Prisma
- DB: PostgreSQL (Node/Edge 정규화)
- 인증: JWT (이메일 / 비밀번호)

---

# 설계 하이라이트 & 안정성

- 정규화 저장: 캔버스를 Node/Edge 테이블로 → 노드 단위 동기화·락·휴지통·로그
- Socket.io 직접 구현 + CollabAPI 추상화 → 막혀도 Liveblocks로 전환(데모 안전)
- 권한 양면 가드: REST + Socket 양쪽 서버 검증 (소유자/에디터/뷰어 3단계)
- 휴지통(복구·영구삭제) + 활동 로그로 실수 방지·변경 이력 관리

직접 구현 + 차선책 추상화로 난이도와 데모 안정성을 동시에

---

# 일정 · 역할 · 마무리

- 4주: 인증·캔버스 기본 → MD 노드·저장·휴지통 → 실시간 협업(하이라이트) → 활동 로그·통합·발표
- 3인: BE 백엔드(소켓+REST+도메인) · F1 캔버스/실시간 · F2 셸/콘텐츠
- 기대 효과: 비어 있던 협업 경험 제공 + 풀스택 직접 설계·구현 + 데모 안정성·기술 난이도 동시 확보
